Multi-repo workspaces,

Create isolated development environments across all your repos using git worktrees. Each workspace gets its own branches, VS Code config, and lifecycle hooks.

Features

Everything you need for
multi-repo development.

Stop juggling branches across repos. Create a workspace, do your work, tear it down.

⚡

Instant workspaces

One command creates worktrees for every repo in your group, with a VS Code workspace file ready to open.

⇶

Fully parallel

Clones, syncs, pulls, pushes, and hooks run concurrently across all repos. Built in Go for speed.

🔗

Short repo names

Set your git org once. Use my-app instead of the full URL. GitHub, GitLab, Bitbucket, or self-hosted.

⚙

Lifecycle hooks

Auto-run pnpm install or any setup command after project creation. Global, group, or per-repo.

📍

Context-aware

hk pull, hk push, hk status auto-detect your project from the current directory.

↻

Self-updating

Checks for updates daily. hk update upgrades in place with no sudo after initial install.

Quick Start

Up and running in under a minute.

Install, create a config, sync your repos, and start a new workspace. Both hike and hk are installed automatically.

~/projects
$ hk init
Created hike.yaml

$ hk sync
Syncing group "platform" (3 repos)...
  my-app: cloned
  shared-lib: cloned
  api: cloned

$ hk platform
Creating "platform-bold-cedar"...
Project "platform-bold-cedar" created.

$ cd platform-bold-cedar
$ hk status
  my-app       branch: platform-bold-cedar
  shared-lib   branch: platform-bold-cedar
  api          branch: platform-bold-cedar
● ● ● hike.yaml 
git:
  org: your-org

hooks:
  onCreate: pnpm install

groups:
  platform:
    default: true
    repos:
      - my-app
      - shared-lib
      - repo: api
        branch: develop

  marketing:
    aliases: [mktg]
    repos:
      - website
      - cms
Configuration

One YAML file. That's it.

Define your git org, groups, repos, and hooks. Short names expand automatically. Override branch, name, or hooks per repo.

Commands

Simple, memorable commands.

Every command works with hike or hk.

hk platform Create workspace
hk sync Clone & sync repos
hk pull Pull all repos
hk push Push all repos
hk status Git status
hk delete Remove workspace
hk list List projects
hk update Self-update
